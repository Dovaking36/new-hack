# GigaChat Telegram Bot (MVP)

Telegram‑бот с интеграцией нейросетевой модели GigaChat от Сбера. Бот умеет отвечать на вопросы, анализировать историю чата, распознавать голосовые сообщения, читать Excel и PDF‑файлы (включая OCR для сканов), обрабатывать изображения и отправлять уведомления.

> ⚠️ **Важно:** проект является **MVP (Minimum Viable Product)**.  
> - Нет СУБД – история хранится в JSON‑файлах на диске.  
> - Нет RAG (Retrieval‑Augmented Generation) – поиск по истории примитивный (по ключевым словам или за последние дни).  
> - Нет масштабирования, кеширования, полноценной обработки ошибок для production.  
> Проект демонстрирует интеграцию GigaChat с Telegram и базовые возможности агента с инструментами.

## 🚀 Возможности

| Функция | Описание |
|---------|----------|
| 🤖 **Общение с агентом** | Поддержка диалога с памятью в рамках сессии. |
| 📊 **Чтение Excel** | Извлечение данных из `.xlsx`, `.xls`, `.ods`. |
| 📄 **Чтение PDF** | Извлечение текста + OCR сканированных страниц через GigaChat Vision. |
| 🎤 **Голосовые сообщения** | Транскрипция речи в текст. |
| 🖼️ **Изображения** | Описание содержимого и распознавание текста (OCR) через Vision. |
| 📜 **История чата** | Сохранение всех сообщений в JSON, возможность экспорта в `.txt` и поиск по истории. |
| 🔔 **Уведомления** | Бот может отправлять сообщения в чат по запросу. |
| ⏰ **Анализ событий** | Периодический анализ истории (каждые 25 минут) для выявления упоминаний о встречах/дедлайнах и отправки напоминаний. |
| 🧮 **Калькулятор** | Сложение чисел через инструмент. |

## 🛠️ Технологии

- **Python 3.11+**
- **aiogram 3.x** – работа с Telegram Bot API
- **LangChain** – фреймворк для создания агента и инструментов
- **GigaChat API** – языковая модель, Vision, транскрипция
- **pandas** – чтение Excel
- **PyMuPDF (fitz)** – извлечение текста из PDF
- **asyncio** – асинхронная обработка

## 🤖 Команды бота

| Команда | Описание | Доступность |
|---------|----------|-------------|
| `/bot [текст]` | Задать вопрос агенту GigaChat (даже в группах без упоминания) | везде |
| `/get_history` | Экспортировать всю историю чата в `.txt` и отправить файлом | только группы |
| `/stats` | Показать статистику сообщений (всего, дней, участников) | только группы |
| `/help` | Справка по командам и возможностям | везде |
| `/start` | Приветственное сообщение | везде |

**В группах** бот также отвечает, если его упомянуть (`@bot_username`) или написать слово «бот» в начале сообщения.  
**В личных сообщениях** отвечает на любой текст автоматически.

## 🧰 Инструменты агента (что умеет вызывать модель)

Агент сам решает, какой инструмент использовать (на основе системного промпта). Доступны:

- `summator(a, b)` – сложение двух чисел.
- `get_chat_id()` – узнать ID текущего чата.
- `send_notification(text)` – отправить сообщение в чат.
- `read_chat_history(limit, days, search)` – получить историю сообщений.
- `transcribe_audio(file_id)` – распознать голосовое.
- `read_excel_file(file_id, sheet_name)` – прочитать Excel.
- `read_pdf_file(file_id, page_start, page_end)` – прочитать PDF (с OCR).

Примеры запросов к агенту:
- «Сколько будет 15 + 27?»
- «Отправь уведомление: встреча в 18:00»
- «О чём мы говорили вчера?»
- «Найди в истории сообщения про дедлайн»
- «Прочитай последний PDF, страницы 2–5»
- «Что в этом Excel?»

## ⚠️ Ограничения MVP

| Ограничение | Пояснение |
|-------------|-----------|
| **Нет СУБД** | История хранится в JSON‑файлах (по одному на день). При большом количестве сообщений чтение/запись становится медленным. |
| **Примитивный поиск** | Поиск по истории – только по ключевому слову или за последние N дней. Нет векторного индексирования (RAG). |
| **Память сессии** | История диалога с агентом хранится в оперативной памяти (`session_store` в `agent.py`). При перезапуске бота память сбрасывается. |
| **Ограничение длины** | Текст из Excel/PDF обрезается до 15–20 тыс. символов, чтобы не превышать контекст модели. |
| **OCR только через API** | Распознавание текста на сканах PDF выполняется вызовом GigaChat Vision – это платно и имеет rate limits. |
| **Однопоточность** | Бот работает в одном потоке asyncio, тяжёлые операции могут блокировать ответы другим пользователям (хотя они вынесены в `to_thread`). |
| **Нет аутентификации** | Бот доступен любому, кто найдёт его токен или добавит в чат. |
